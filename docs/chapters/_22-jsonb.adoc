== JSON-B
JSON-B is the standard JSON binding specification in Jakarta EE. Spring Boot uses Jackson by default. Both support annotations for field names, null values, formatting and serialization rules, but differ in details and default behavior.
=== Annotations and mapping
JSON-B annotations like `@JsonbProperty` can be transferred to `@JsonProperty` in most cases. `@JsonbDateFormat`, `@JsonbTypeAdapter` and visibility rules for properties require special attention.
.Example: BEFORE (JSON-B)
[source, java]
----
import javax.json.bind.annotation.JsonbProperty;
import javax.json.bind.annotation.JsonbDateFormat;

public class ShipmentSnapshot {

    @JsonbProperty("tracking_id")
    private String trackingId;

    @JsonbDateFormat("yyyy-MM-dd")
    private LocalDate departureDate;
}
----

.Example: AFTER (Jackson)
[source, java]
----
import com.fasterxml.jackson.annotation.JsonProperty;
import com.fasterxml.jackson.annotation.JsonFormat;

public class ShipmentSnapshot {

    @JsonProperty("tracking_id")
    private String trackingId;

    @JsonFormat(pattern = "yyyy-MM-dd")
    private LocalDate departureDate;
}
----

The migration should validate the actual JSON contract behavior with tests, especially if clients rely on specific field names, date patterns or null value handling.
