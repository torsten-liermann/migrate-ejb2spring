== Email
Jakarta Mail is often integrated into EJB applications via `Session` objects from JNDI. Spring Boot offers a simplified integration with `JavaMailSender`, which is configured via properties. The migration affects both injection and configuration.
=== JNDI session to JavaMailSender
.Example: BEFORE (EJB)
[source, java]----
import javax.annotation.Resource;
import javax.mail.Session;

public class NotificationService {

    @Resource(lookup = "java:app/mail/Notification")
    private Session mailSession;
}
----

.Example: AFTER (Spring Boot)
[source, java]----
import org.springframework.mail.javamail.JavaMailSender;
import org.springframework.stereotype.Service;

@Service
public class NotificationService {

    private final JavaMailSender mailSender;

    public NotificationService(JavaMailSender mailSender) {
        this.mailSender = mailSender;
    }
}
----

The configuration is done via `spring.mail.*` in `application.properties`. Security-relevant values ​​such as passwords should be integrated via secrets or vaults.
