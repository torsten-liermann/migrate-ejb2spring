= EJB 3.x / Jakarta EE zu Spring Boot Migration
:toc: left
:toclevels: 3
:sectnums:
:icons: font
:source-highlighter: highlight.js
:experimental:

[.lead]
Vollständige Referenz für die Migration von Enterprise JavaBeans (EJB) 3.x und Jakarta‑EE‑Konstrukten nach Spring Boot. Jeder Abschnitt zeigt VORHER (EJB/Jakarta EE) und NACHHER (Spring Boot) mit konkreten Code‑Beispielen.

[IMPORTANT]
====
Spring Boot stellt keinen EJB‑Container bereit. Damit entfallen EJB‑spezifische Laufzeitdienste wie Pooling, Passivierung/Aktivierung bei Stateful Beans, containerverwaltetes Remoting, `@Lock`‑Semantik, MDB‑Runtime und JCA‑Resource‑Adapter. Die Migration ist daher ein Wechsel des Programmiermodells und nicht nur ein Annotationstausch.
====

=== Migrationskategorien

[cols="1,2,2"]
|===
| Kategorie | Charakteristik | Beispiele

| API‑nah (mechanisch)
| Spring nutzt dieselben oder ähnliche APIs. Migration ist weitgehend mechanisch (Namespace‑Wechsel, Annotation‑Mapping).
| JPA, Bean Validation, JSON-B, Servlet, JAX-RS → Spring MVC

| Container‑Semantik (Architektur)
| Der EJB‑Container bietet Laufzeitverhalten, das Spring nicht repliziert. Erfordert Architekturentscheidungen.
| Stateful Beans, Singleton-Locking, MDB, Timer, JCA, Remoting
|===

[NOTE]
====
In diesem Projekt werden Beispiele mit Konstruktorinjektion gezeigt. Field Injection ist in den Beispielen nicht verwendet.
====

Konfigurationsreferenz: `docs/research/project-configuration-yaml.md`.
